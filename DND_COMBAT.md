# D&D 风回合制战斗系统 —— 规则设计表 v2（待评审）

> **本文是 v2，废弃了 v1 的"四类型"方案。** 新模型核心三件事：
> 1. **属性点分配**：没有类型，每只精灵有 STR/DEX/CON，升级自由加点、可洗点。
> 2. **技能池选学**：升级时从一个小而精（6–8 个）的技能池里选学技能，像 D&D 选 feat。
> 3. **真·回合制手动战斗**：每回合玩家自己选「普攻 / 放哪个技能」，不再自动观战。

## 一、属性系统

每只精灵 3 个核心属性（精简自 5e 的六维，够用且好懂）：

| 属性 | 含义 | 影响的战斗数值 |
|------|------|--------------|
| **STR 力量** | 物理强度 | 命中调整 `+STR_mod`、伤害调整 `+STR_mod` |
| **DEX 敏捷** | 灵巧 | AC `10 + DEX_mod`（越高越难被命中）、先攻 `1d20 + DEX_mod` |
| **CON 体质** | 耐力 | 最大 HP `(8 + CON_mod) × level` |

**调整值公式（5e 标准）**：`mod = floor((属性 - 10) / 2)`
（10→+0, 12→+1, 14→+2, 16→+3, 18→+4, 20→+5；8→−1, 6→−2）

**熟练加值 PRO**（随等级，加进命中骰）：Lv1-4 = +2，5-8 = +3，9-12 = +4，13-15 = +5。

### 加点规则
- **初始**：每只精灵有不同的**天赋起点**（见第二节），三属性总和相同（公平），分布不同。
- **每升一级**：获得 **+2 属性点**自由分配（任意加到 STR/DEX/CON），单项属性上限 20。
- **洗点**：可随时重置，把已分配的点全部退回重新分（MVP 阶段免费、无 CD；以后可加代价）。
- 升级还会触发**选技能**（见第四节）。

## 三、核心战斗判定（每次攻击）

```
1. 命中：攻击者掷 1d20 + STR_mod + PRO  vs  防御者 AC(10 + DEX_mod)
   - 自然 20 → 暴击命中（伤害骰数量翻倍，必中）
   - 自然 1  → 大失败（必落空）
   - 总和 ≥ AC → 命中；否则落空（= 被"闪开"，DEX 高自然难被打中）
2. 伤害：基础伤害骰 1d6 + STR_mod（普攻）
   - 暴击：伤害骰**数量翻倍**（1d6 → 2d6），调整值不翻倍（5e 正统）
   - 技能会改伤害骰（见第四节）
3. 扣血：HP -= max(0, 伤害 - 减伤)
```

**事件流带骰子明细**，前端日志渲染成跑团风：
```
Charmander 普通攻击
🎲 命中 1d20+5 = [14]+5 = 19  vs AC 12 → 命中
🎲 伤害 1d6+3 = [5]+3 = 8
Onix 受到 8 点伤害
```

## 四、技能池（小而精，8 个）+ 选学规则

升级时（每级）可从技能池**选学 1 个**还没学的技能。技能学了就常驻可用，
**战斗中每回合手动选**「普攻 / 放某个已学技能」。部分技能有 CD（冷却回合）。

| # | 技能名 | 效果 | CD |
|---|--------|------|----|
| 1 | **英勇打击** | 本次攻击伤害骰 1d6→2d6，命中 +2 | 1 |
| 2 | **生命汲取** | 本回合不攻击，回复 `1d8 + CON_mod` HP | 2 |
| 3 | **石化表皮** | 接下来 2 回合受到伤害 −`1d6`（掷一次定减免量）| 3 |
| 4 | **眩晕突袭** | 攻击命中后，对方掷 `1d20+CON_mod`，<13 则下回合昏迷（无法行动）| 3 |
| 5 | **疾风连击** | 本回合攻击 2 次（各自独立命中/伤害骰）| 2 |
| 6 | **蓄力重击** | 本回合不动，下回合攻击伤害骰 1d6→3d6 且必命中 | 3 |
| 7 | **精准瞄准** | 本次攻击改投 2d20 取高（优势骰），易暴击 | 1 |
| 8 | **护盾格挡** | 本回合 AC +5，并反弹 `1d4` 伤害给攻击者 | 2 |

> 技能**不绑属性**，任何 build 都能学任何技能；但效果吃属性（如生命汲取吃 CON、
> 重击吃 STR 的伤害调整），所以 build 仍然影响技能强度。这鼓励「属性 + 技能」搭配。
> 8 个里玩家整个养成期（Lv1→15 共 14 次升级）能学满，但**学的顺序**和**加点配合**是策略。

## 五、回合制战斗流程（状态机，替代自动回放）

```
战斗开始：双方各掷先攻 1d20+DEX_mod，高者先动（平手再掷）
每一轮 round：按先攻顺序，每方依次行动：
  → 引擎暴露「当前可选动作列表」(普攻 + 未在CD的已学技能)
  → 等玩家（或 AI / 对手）选一个动作
  → 引擎结算这一步，产出带骰子明细的事件，推进状态
  → CD 递减、持续效果（眩晕/石化/蓄力）结算
检查存活，一方 HP≤0 → 战斗结束
```

**引擎形态**：从 v1 的"一次算完整场纯函数"改为**确定性状态机**：
- `createBattle(a, b, seed) → BattleState`
- `legalActions(state) → Action[]`
- `applyAction(state, action) → { state, events }`（确定性：同 state+action+内部 RNG 游标恒等输出）

这套天然适配：
- **单机 PvE**：敌人动作由简单 AI 选
- **联机 PvP**：服务端持有权威 state，两端各自选动作发给服务端，服务端结算后广播事件
- **可测**：给定动作序列，断言事件/胜负

# v3 增补：BG3 风消耗体系 —— 用「法术位 + 吸血附加」替换 CD（待评审）

> 决策（2026-06-07）：走 BG3 风资源位 + 行动经济，**彻底换掉 CD**；
> 生命汲取从独立技能改为「吸血附加」。顺带修两个 UX：技能说明直接显示、
> 资源不足/不可用的技能要灰显而非消失。

## v3-1 为什么换掉 CD
CD 是 MOBA/MMO 机制，和"D&D/BG3 风"定位不搭。BG3 的精髓是**两层约束**：
- **行动经济**：一回合能做几件事（action/bonus/reaction）
- **长线资源**：法术位（spell slots），整场/长休制，决定"大招能放几次"

CD 让强力技能"循环轰炸"（如吸血每 2 回合无脑回血 → 赖皮）。资源位让大招**有限且需规划**。

## v3-2 裁剪到 1v1 对战的规则
BG3 的三层行动经济对自走式 1v1 太重。**简化为单层行动 + 法术位资源**：

### 行动经济（单层）
每回合你做**一个主要动作**：普攻 / 放一个技能。不引入 bonus action 的复杂度。
（保留扩展空间：以后可加"每回合可追加一个标记为 bonus 的小技能"。）

### 法术位（spell slots）—— 核心长线资源
- 每场战斗开始，按等级获得若干**法术位**：`slots = 1 + floor(level / 4)`（Lv1→1，Lv4→2，Lv8→3，Lv12→4，Lv15→4）。
- 技能分两类：
  - **戏法/普通技能（无消耗）**：可无限放（如精准瞄准、护盾格挡这类小技能）
  - **法术（消耗 1 法术位）**：强力技能（英勇打击、疾风连击、蓄力重击、眩晕突袭）
- 法术位**整场不回**（对应 BG3 的"长休才回"，但一场战斗内不长休）。打光了只能靠普攻和戏法。
- 没有 CD。资源用尽即不可用（界面灰显"无法术位"）。

> 这把"技能强度"从"等多久"变成"还剩几次"，更有规划感，也天然解决赖皮：
> 强力技能就那么几发，必须挑时机。

### 技能池按消耗重新分类（8 个）
| 技能 | 消耗 | 说明 |
|------|------|------|
| 普通攻击 | 无 | 1d6+STR，可吸血附加（见 v3-3）|
| 精准瞄准 | 无（戏法）| 优势命中，鼓励普攻流也有操作 |
| 护盾格挡 | 无（戏法）| 防御姿态，AC+5 反弹 |
| 石化表皮 | 无（戏法）| 减伤姿态 |
| 英勇打击 | 1 法术位 | 强化攻击 |
| 疾风连击 | 1 法术位 | 双段攻击 |
| 蓄力重击 | 1 法术位 | 必中重击 |
| 眩晕突袭 | 1 法术位 | 控制 |

> 戏法（无消耗）也可加"每回合只能用其中一个"之类的轻约束，避免无限叠防御。MVP 先不加。

## v3-3 吸血改为「吸血附加」（lifesteal）
- 删掉独立的"生命汲取"技能。改为一个**属性/被动**：造成伤害时回复 `floor(dealt × lifestealRate)`。
- lifestealRate 来源（二选一，待定，见待拍板）：
  - (A) 作为一个**可学的被动技能**"吸血"，学了之后所有普攻/技能伤害回 25%
  - (B) 与某属性挂钩（如 CON 高给少量吸血）
- 关键：**只有打中才回血**，旱涝不再保收；高 AC 对手能让吸血流断奶。

## v3-4 引擎/UI 改动
- `Fighter` 去掉 `cooldowns`，加 `slots: number`（当前剩余法术位）+ `maxSlots`。
- `skills.ts`：每个技能加 `cost: 0 | 1`（法术位）字段；去掉 `cooldown`。
- `legalActions`：返回**全部已学技能**，但每个带 `usable: boolean` + 理由（资源不足/昏迷）；
  界面据此灰显并显示"剩 N 法术位 / 无法术位"。（或保持只返回可用，另出一个 `allActions(state)` 给 UI 显示全量+状态。）
- 伤害管线：命中造成伤害后，若攻击方有吸血，结算回血事件。
- `applyAction`：技能消耗法术位；移除 CD 递减逻辑。
- client：BuildEditor 直接显示技能说明（不靠 tooltip）；战斗面板显示法术位余量、灰显不可用技能。

## v3-5 已拍板（2026-06-07）
1. 法术位 `1 + floor(level/4)`，整场不回 → 用。
2. 技能分类按 v3-2 → 用（4 耗位 / 3 免费戏法 + 普攻）。
3. **吸血挂钩 CON**：被动，不占技能位。公式：`lifestealRate = max(0, CON_mod) × 5%`
   （CON14→+2→10% 吸血，CON20→+5→25%）。只有打中造成伤害才回血，
   回血量 = `floor(dealt × rate)`。删掉独立的"生命汲取"技能。
4. 戏法暂不加"每回合一个"约束（MVP）。

> 注：删了"生命汲取"后技能池剩 7 个（普攻另算）。即英勇打击/疾风连击/蓄力重击/眩晕突袭(耗位)
> + 精准瞄准/护盾格挡/石化表皮(戏法)。吸血变成 CON 被动，不在技能池里。
