# 美术与动画：现状 + 剩余路线

> 原始版本是"多状态图/动图"的决策方案文档；核心思路（**程序化动画 + 静态姿势图**，
> 把"动图"需求降维成"代码动效 + 每角色几张图"）已被采纳并基本落地。
> 本文现在是**状态清单**：记录做到哪了、还剩什么。

---

## 已落地 ✅

### 地基：事件时间轴回放（原 Step A）
- 引擎事件流不再瞬时结算，**入队按节拍逐个折叠**进显示态（`useBattle.ts` + `playback.ts`），
  可调速 1x / 2x / 瞬间；AI 推进等回放清空后再走。

### 资产管线（原 Step C/D，已超出原计划）
- `npm run art`（`art/process.mjs`）：alpha 包围盒裁切 → 等比居中 → 1024 WebP。
- **多姿势**：`<Id>.attack/.hit/.downed.png` → 独立姿势 sprite；同角色多姿势同画布尺寸时
  用**联合包围盒**统一裁切（锚点对齐，切图不跳位）。
- **背景**：`art/raw/bg/` → cover 裁切 1760×880 + 压暗/去饱和/轻模糊 + `manifest.json`
  （游戏内下拉框自动列出）。
- `--flatten-bg[=容差]` 纯色去背，已透明的图逐文件自动跳过。
- 12 角色 idle + attack + downed 已接入；**缺哪张回退哪张**（姿势→idle→圆形占位）。
- 生图侧：[`art/PROMPTS.md`](art/PROMPTS.md) 提供 12 角色 prompt 包 + 编辑式一致性锁定模板。

### 状态动画（原 Step B 的主体）
- **姿势切换**：按回放事件驱动——出手→attack 图、被命中→hit 图、倒地/死亡→downed 图
  （死亡叠灰度；缺 downed 图回退"idle + 旋转躺倒"）。
- **突进**：单体攻击时攻击者 rAF 缓动滑到目标面前出手、回合结束归位；AOE/自身原地。
- **画质**：HiDPI（devicePixelRatio）渲染、脚下椭圆阴影 + 柔和投影（角色与背景分离）。

---

## 剩余 ⬜（按性价比排序）

1. **跳字**：伤害/治疗数字从角色头上飘出（事件里有完整数值，纯渲染活）。
2. **受击闪白 / 暴击放大**：命中瞬间目标闪一下；nat20 额外缩放脉冲 + 更大跳字。
3. **死亡表现**：淡出 + 缩小消失（当前是灰度 + 半透明常驻）。
4. **hit 姿势图补齐**：生图侧产出 12 张 `.hit.png`（管线/表现层已就绪，缺图回退 idle）。
5. （可选）**高光特效 / 局部序列帧**（原 Step E）：暴击/签名技能的专门特效。

---

## 设计约束（不变）

- 全部动效在 `packages/client`，**不碰 shared**：引擎确定性与单测不受影响。
- 资产是可选增强：任何缺图都有回退链，不阻塞开发与游玩。
- 真·动图（animated WebP/GIF）仍不做——代码动效 + 静态姿势图的观感性价比更高。
